# 开发路线图

## 当前已完成

- 项目分层骨架：domain / application / infrastructure / presentation。
- NGA 保存页 HTML 解析适配器。
- 标准 `ThreadSnapshot` 输出。
- 目录内多页保存 HTML 合并。
- 引用与回复关系候选抽取。
- 基础历史分析报告。
- 规则型市场实体与线索抽取。
- 规则型观点候选抽取。
- 规则型观点链候选构建，支持明确证据与推断关联分层。
- 规则型隐晦表达候选抽取，覆盖历史延续、隐含对象、条件确认和风险控制短语。
- 数据库、LLM、检索、采集器应用端口。
- 本地文件仓库实现。
- 导入、分析、持久化的一体化应用入口。
- 任务记录与导入任务入口。
- 本地历史证据检索索引。
- 新发言语境还原 MVP。
- 无依赖 HTTP API 入口。
- OpenAPI 契约与 API 文档。
- 无依赖 Web 工作台原型。
- Markdown 报告渲染。
- 论坛适配器注册表。
- Node 内置单元测试。

## 近期优先级

1. 作者历史报告产品化：观点地图、观点链时间线、隐晦表达候选和高价值回帖。
2. LLM 实体抽取增强：在规则词典基础上识别隐晦简称和上下文指代。
3. LLM 观点抽取增强：识别更隐晦的态度变化和观点延续。
4. 证据链：每个结论引用楼层、作者、时间、原文片段，并区分明确依据与模型推测。
5. 前端工作台：作者看板、主题时间线、报告页。
6. 实时提醒：在历史观点链基础上检测重要变化并发送提醒。

## 资源接口预留

- PostgreSQL：结构化数据、报告索引、任务状态。
- 对象存储或本地归档：原始 HTML、附件、解析快照。
- 向量库：语境召回和相似观点检索。
- LLM Provider：实体抽取、观点抽取、新发言解读。
- 任务队列：多页采集、增量分析、报告刷新。
