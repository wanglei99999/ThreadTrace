# 开发路线图

## 当前已完成

- 项目分层骨架：domain / application / infrastructure / presentation。
- NGA 保存页 HTML 解析适配器。
- 标准 `ThreadSnapshot` 输出。
- 基础历史分析报告。
- Markdown 报告渲染。
- 论坛适配器注册表。
- Node 内置单元测试。

## 近期优先级

1. 多页合并：把同一主题的多个保存页合并为一个完整主题快照。
2. 原始数据仓库：将 HTML、解析结果、报告分层保存，支持重跑。
3. 实体抽取接口：先做规则词典，再接 LLM。
4. 观点抽取接口：定义态度、周期、条件、证据、置信度结构。
5. 证据链：每个结论引用楼层、作者、时间、原文片段。
6. Web API：用同一应用层暴露 HTTP 入口。
7. 前端工作台：作者看板、主题时间线、报告页。

## 资源接口预留

- PostgreSQL：结构化数据、报告索引、任务状态。
- 对象存储或本地归档：原始 HTML、附件、解析快照。
- 向量库：语境召回和相似观点检索。
- LLM Provider：实体抽取、观点抽取、新发言解读。
- 任务队列：多页采集、增量分析、报告刷新。
