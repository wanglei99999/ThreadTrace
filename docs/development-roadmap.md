# 开发路线图

## 当前已完成

- 项目分层骨架：domain / application / infrastructure / presentation。
- NGA 保存页 HTML 解析适配器。
- 标准 `ThreadSnapshot` 输出。
- 目录内多页保存 HTML 合并。
- 引用与回复关系候选抽取。
- 基础历史分析报告。
- 规则型市场实体与线索抽取。
- 规则型观点候选抽取。
- 数据库、LLM、检索、采集器应用端口。
- 本地文件仓库实现。
- Markdown 报告渲染。
- 论坛适配器注册表。
- Node 内置单元测试。

## 近期优先级

1. PostgreSQL 文件外实现：将 HTML、解析结果、报告分层保存，支持重跑。
2. LLM 实体抽取增强：在规则词典基础上识别隐晦简称和上下文指代。
3. LLM 观点抽取增强：识别更隐晦的态度变化和观点延续。
4. 证据链：每个结论引用楼层、作者、时间、原文片段。
5. Web API：用同一应用层暴露 HTTP 入口。
6. 前端工作台：作者看板、主题时间线、报告页。

## 资源接口预留

- PostgreSQL：结构化数据、报告索引、任务状态。
- 对象存储或本地归档：原始 HTML、附件、解析快照。
- 向量库：语境召回和相似观点检索。
- LLM Provider：实体抽取、观点抽取、新发言解读。
- 任务队列：多页采集、增量分析、报告刷新。
