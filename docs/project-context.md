# 项目上下文

## 产品定位

语脉 ThreadTrace 是一个论坛观点追踪与语境还原系统。当前场景聚焦于某个资深股民作者在 NGA 主题帖中的长期讨论：系统解析历史楼层、作者发言、回帖追问、引用关系和关键证据，生成可回溯的历史分析报告。

它的目标不是预测涨跌，也不是生成买卖建议，而是降低用户理解论坛隐晦表达的成本。例如作者说“昨天那个方向，今天算验证了，但别追”，系统应尽量回答：

- “那个方向”最可能指向哪只股票、哪个板块或哪个事件；
- 这个判断延续了哪些历史楼层；
- 作者当前态度是否从关注、验证、谨慎或放弃之间发生变化；
- 哪些是明确证据，哪些是模型推测；
- 结论置信度来自哪里。

## 第一阶段 MVP

第一阶段优先做历史分析，不做实时提醒。

1. 输入一个论坛主题帖的本地 HTML 保存页。
2. 解析帖子标题、论坛、tid、分页信息、楼层、作者、uid、时间、正文、链接和推荐值。
3. 转换为统一的 ThreadSnapshot 数据结构。
4. 生成基础历史报告：楼层概况、主楼作者、作者参与情况、外链线索、高价值楼层候选、噪声楼层候选。
5. 为后续 LLM 抽取观点、实体、证据链留下接口。

## 后续阶段

第二阶段：多页、多主题合并，建立同一作者的长期档案。

第三阶段：接入 LLM，抽取实体、观点、条件、态度、时间周期和证据片段。

第四阶段：接数据库、向量检索和全文检索，支持历史语境召回。

第五阶段：网页工作台和定时提醒，展示作者看板、观点时间线和新发言解读报告。
